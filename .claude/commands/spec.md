Invoke the oh-my-zz:idea-to-spec-and-plan skill.

Use the supplied idea document, or locate a clearly matching draft under `docs/ideas/`. Clarify only material gaps and establish what reproducible evidence the user accepts as proof of completion.

Generate and save a baselined specification and one repository-grounded candidate plan from that exact baseline. Use milestones only to group the complete plan, include a reviewable test execution contract, and present the two artifacts together for approval; do not implement code or start plan review automatically. If the user explicitly requests review, invoke `plan-review` separately in the main agent.
